## 3. Brand Personality and Writing

MathsTutorHelp should feel:

- Clear and calm
- Trustworthy
- Professional but approachable
- Practical and parent-friendly
- Focused on real teaching

### Writing style

Use plain, confident UK English. Keep paragraphs short and make the next step obvious.

Use:

- Maths, not “math”
- Tuition, tutor, learner or student
- Personalised
- Clear explanations
- Guided practice
- Exam-style questions
- Steady progress

Avoid:

- Hype or pressure
- “Best tutor”, “guaranteed grades” or “100% success”
- American spelling
- Dense paragraphs
- Generic education slogans
- Mixing one-to-one tuition prices with membership prices

### Approved calls to action

| Situation                     | Button text              |
| ----------------------------- | ------------------------ |
| General or one-to-one enquiry | Book a Free Consultation |
| GCSE group class enquiry      | Register Your Interest   |
| Direct contact                | Call or WhatsApp         |
| Existing learners             | Student Login            |

Registering interest does not confirm a place and does not take payment. Do not describe the interest form as a completed booking.

### Useful enquiry details

Ask for only the information needed to recommend suitable support:

- Year group or qualification
- Current grade or working level
- Target grade or learning goal
- Topics needing support
- Preferred format: in person, online or group
- General availability when relevant

| Footer            | Ink navy      | `#0B1F3A` |

Use navy for headings and strong brand areas. Use teal for labels and supporting details. Reserve yellow for the main action or a small important badge.

Avoid purple gradients, neon colours and busy rainbow palettes.

### Logo

The compact logo is the **MTH monogram**:

- White `MTH` letters
- Navy rounded square
- Teal underline
- No gradient, shadow or extra symbol

Always include all three letters. Do not replace the teal underline with yellow; yellow belongs to calls to action.

Use the monogram for favicons, social profile images and small square placements. Use the full **MathsTutorHelp** name where there is enough horizontal space.

For a full wordmark, use navy for **Maths** and teal for **TutorHelp**. Keep clear space around it and do not stretch, rotate or recolour it.

Website logo files:

- `src/assets/favicons/favicon.svg`
- `src/assets/favicons/favicon.png`
- `src/assets/favicons/apple-touch-icon.png`

### Typography

- Use Inter
- Use bold or extra-bold headings
- Use regular or medium body text
- Keep normal body text at 16px or larger
- Use comfortable line spacing
- Avoid decorative, handwritten and serif display fonts

### Website style

- Generous white space
- White cards with rounded corners
- Fine light-grey borders and soft shadows
- Soft-blue information panels
- Simple line icons
- One clearly dominant action per section
- Responsive layouts that stack on small screens
- Touch targets around 44px or larger

Avoid heavy shadows, glass effects, excessive animation and tightly packed layouts.

### Images

Prefer genuine, well-lit photographs of Mr Sheraz teaching, maths work or real learning spaces. Use natural colours, rounded corners and restrained shadows.

Do not alter Mr Sheraz's face or invent a different tutor. Treat people in campaign artwork as illustrative students, not documentary classroom photography.

## 5. Standard Page Structure

Use this order for important landing pages and campaigns:

1. Short category label
2. Clear benefit-led heading
3. One supporting sentence
4. One primary action and, if needed, one secondary action
5. Trust line
6. Genuine tutor, classroom or relevant course image
7. Service details
8. Frequently asked questions
9. Final call to action

### Accessibility rules

- Keep body text at 16px or larger
- Use strong colour contrast
- Do not place important text over a busy photograph
- Do not rely on colour alone to explain meaning
- Use descriptive button text instead of “Click here”
- Keep headings in a logical order
- Give every meaningful image useful alternative text
- Make controls usable by keyboard and touch

## 8. Safe Editing and Publishing

Run commands from:

```bash
cd /home/sheraz/code/mathstutorhelp
```

Useful commands:

```bash
npm run dev       # open a local development version
npm run check     # check Astro, ESLint and formatting
npm run build     # create the production website
npm run preview   # preview the production build
```

Before publishing:

1. Check phone, WhatsApp and enquiry links.
2. Check all prices.
3. Check current class dates, times, venue and availability.
4. Confirm Foundation and Higher grade wording.
5. Confirm images are genuine or clearly illustrative.
6. Run `npm run check`.
7. Run `npm run build`.
8. Test the homepage, contact page, pricing page and enquiry buttons on mobile.

### Important technical rules

- The website uses HTTPS: `https://mathstutorhelp.com`.
- Use `getPermalink()` for internal page links.
- Use `getAsset()` for files in `public/`.
- Shared responsive images belong in `src/assets/` and `src/data/images.ts`.
- Do not manually edit `dist/`, `.astro/` or `node_modules/`.
- Hai-BL blocks iframe embedding, so link to its form rather than embedding it.
